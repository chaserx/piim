# piim

A CLI tool that detects and masks personally identifiable information (PII) in PDF documents. Handles both selectable-text and scanned/image-based PDFs via OCR fallback.

## Motivation

Sharing PDF documents often requires removing sensitive information first. Existing redaction tools are either manual (requiring you to highlight each region) or locked behind expensive enterprise software. piim automates the process: point it at a PDF and it finds names, addresses, phone numbers, emails, credit card numbers, and bank account numbers, then masks them with black boxes or realistic fake data.

## How It Works

piim processes each PDF through a four-stage pipeline:

```
PDF → Extract → Detect → Mask → Export
```

1. **Extract** — Pulls text with bounding-box coordinates from each page. Tries native PDF text first; falls back to OCR (EasyOCR) for scanned/image pages.
2. **Detect** — Runs extracted text through Microsoft Presidio to identify PII entities, then maps character offsets back to precise page coordinates.
3. **Mask** — Either draws black rectangles over PII regions (blackbox mode) or replaces them with consistent fake data generated by Faker.
4. **Export** — Saves the redacted PDF with stripped metadata and compression. Supports atomic in-place replacement.

## Usage

```bash
# Basic redaction (black boxes, output to same directory with _redacted suffix)
piim document.pdf

# Fake data replacement
piim document.pdf --mask-type fake

# Process multiple files to a specific directory
piim file1.pdf file2.pdf --output-dir ./redacted/

# In-place redaction (overwrites originals)
piim document.pdf --in-place

# Tune detection sensitivity and see what was found
piim document.pdf --min-confidence 0.7 --verbose
```

### Options

| Flag | Description | Default |
|---|---|---|
| `INPUT` | One or more PDF files | (required) |
| `--mask-type {blackbox,fake}` | Redaction method | `blackbox` |
| `--in-place` | Overwrite original files | off |
| `--output-dir DIR` | Output directory (created if needed) | same as input |
| `--suffix TEXT` | Output filename suffix | `_redacted` |
| `--min-confidence FLOAT` | PII detection threshold (0.0–1.0) | `0.5` |
| `--seed INT` | Faker seed for reproducible fake data | `0` |
| `--verbose` | Log detected entities with scores | off |

`--in-place` is mutually exclusive with `--output-dir` and `--suffix`.

## Architecture

```
piim/
├── cli.py              # Argument parsing and pipeline orchestration
├── models.py           # Shared data types (TextBlock, PiiEntity)
├── extractor.py        # Native text extraction + OCR fallback
├── detector/
│   ├── base.py         # Abstract PiiDetector interface
│   └── presidio.py     # Presidio-based implementation
├── masker.py           # Redaction and fake data insertion
└── exporter.py         # PDF saving with metadata stripping
```

Each stage depends only on the shared data types in `models.py`. The detector uses a pluggable interface (`PiiDetector`) so alternative detection backends can be swapped in.

### Supported PII Types

- Person names
- Locations/addresses
- Phone numbers
- Email addresses
- Credit card numbers
- US bank account numbers

## Limitations

- **English only** — OCR and detection are configured for English text.
- **Mixed pages** — Pages use a binary native-text-or-OCR decision. If a page has selectable text above the threshold, embedded images on that page won't be OCR'd.
- **Sequential processing** — Pages are processed one at a time (~2–5s per page with OCR).
- **Font matching** — Fake data uses Helvetica; it won't visually match the original document's fonts.
